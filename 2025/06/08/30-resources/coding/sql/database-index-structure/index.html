非唯一字段的索引

对于非唯一字段，索引结构会略有不同：

  1. 在索引中存储重复值
  2. 对于每个重复值，存储多个指向不同数据行的指针

例如，假设我们有一个 “年龄” 字段的索引：

Code

其中 18 和 22 表示有多个学生的年龄是 18 或 22。

非唯一字段索引的影响

  1. 存储空间：需要更多空间来存储重复值和额外的指针。
  2. 查询性能：
    • 精确匹配查询可能需要额外步骤来检索所有匹配的行。
    • 范围查询通常不受太大影响。
  3. 更新性能：插入、更新和删除操作可能需要更多时间来维护索引结构。
