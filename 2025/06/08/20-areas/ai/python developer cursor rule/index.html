Write code in python 3.12. Follow best community python best practice.
Please always review your code to see if it follows the guide below and change accordingly.

Coding Guide

  1. Implement a lot of logging and error handling
  2. Handle edge cases, invalid inputs, and unusual scenarios
  3. Optimize performance as much as possible while still meeting original requirements.
  4. Use type hints throughout the code, all variables
  5. Provide very comprehensive docstrings (Google style), in cluding examples in docstring to fulfill the needs of sphinx. every of the function or method.
  6. Whenever possible, do test driven development. pytest should be designed to cover all requirements, and be very fast for running, for fast pace development.

Repository Overview

Project Name: AutoWriterLLM

Description:
A Python tool that helps automatically generate comprehensive book structures and content using Large Language Models (LLMs). It provides functionality for content generation, markdown to PDF conversion, and interactive content creation.

Technology Stack:

  • Programming Language: Python 3.9-3.12
  • Framework: Custom framework built around LLM integration
  • Key Dependencies:
    • anthropic (LLM integration)
    • typer (CLI interface)
    • markdown (content processing)
    • chromadb (vector database)
    • sentence-transformers (embeddings)
    • PyYAML (configuration)

Architecture Overview

The project follows a modular architecture with clear separation of concerns between content generation, context management, and output formatting.

UML Class Diagram

Code

Key Components

Content Generator (ai_content_generator.py)

  • Description: The core component responsible for generating content using LLMs. It handles the interaction with AI models, processes prompts, and structures the generated content.
  • Key Features:
    • Book structure generation
    • Chapter content generation
    • Content processing and formatting
    • Integration with LLM APIs (primarily Anthropic)

Markdown to PDF Converter (markdown_to_pdf_converter.py)

  • Description: Converts markdown content to PDF format with customizable styling and formatting options.
  • Key Features:
    • Markdown parsing and processing
    • PDF generation with customizable templates
    • Support for various styling options

Context Management (context_management.py)

  • Description: Manages the context for content generation, ensuring coherence across different sections and chapters.
  • Key Features:
    • Context storage and retrieval
    • Context updating based on generated content
    • Maintaining consistency across content

Knowledge Graph (knowledge_graph.py)

  • Description: Builds and maintains a knowledge graph to represent relationships between concepts in the generated content.
  • Key Features:
    • Graph construction from content
    • Query capabilities for related concepts
    • Integration with content generation

Review System (review.py)

  • Description: Provides functionality for reviewing generated content and suggesting improvements.
  • Key Features:
    • Content quality assessment
    • Feedback generation
    • Revision suggestions

Learning Path Generator (learning_path.py)

  • Description: Generates customized learning paths based on content and user preferences.
  • Key Features:
    • Path generation based on content structure
    • Customization options
    • Progress tracking

Code Quality Checker (code_quality.py)

  • Description: Checks and improves the quality of code snippets in the generated content.
  • Key Features:
    • Code analysis
    • Improvement suggestions
    • Code formatting

Interactive Module (interactive.py)

  • Description: Provides interactive functionality for content generation and modification.
  • Key Features:
    • Interactive content creation
    • Real-time feedback
    • User-guided content generation

Dependencies

Core Dependencies

  • typer: CLI interface
  • markdown: Markdown processing
  • PyYAML: Configuration management
  • anthropic: LLM API integration
  • tqdm: Progress bars
  • requests: HTTP requests
  • chromadb: Vector database
  • sentence-transformers: Text embeddings
  • torch: Machine learning framework

Development Dependencies

  • bump2version: Version management
  • tox: Testing automation
  • pytest: Testing framework
  • ruff: Linting
  • mypy: Type checking
  • coverage: Test coverage

Configuration

The project uses YAML configuration files (config.yaml) for managing settings:

  • API keys for LLM services
  • Output formatting preferences
  • Content generation parameters
  • Logging settings

Component Interactions

The ContentGenerator serves as the central component, orchestrating interactions between other modules:

  1. It uses ContextManager to maintain coherence across generated content
  2. Leverages KnowledgeGraph to understand relationships between concepts
  3. Employs MarkdownToPDFConverter to produce formatted output
  4. Integrates with ReviewSystem for quality assurance
  5. Works with LearningPathGenerator to create structured learning experiences

The system can be used through both a command-line interface and a GUI (ContentGeneratorGUI), providing flexibility for different user preferences.
